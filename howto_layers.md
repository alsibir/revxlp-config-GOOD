# Способы включения слоёв и условия активации в конфигурации раскладки ZMK revxlp

Нумерация кнопок тамб кластера принята по строке тамб кластера в каждом слое (6 позиций), слева направо: кнопка №1, №2, №3, №4, №5, №6.

Таблица 1 — Способы включения слоёв и условия активации в конфигурации раскладки ZMK revxlp (с указанием кнопок тамб кластера)

| № слоя | Идентификатор слоя в конфигурации                                                | Display name     | Способы включения (прямые триггеры по bindings)                                                                                                                                                                                                                                                                                                 | Примечания и ограничения по текущей раскладке                                                                                                                                                                                                                                                                                                                                |
|--------|----------------------------------------------------------------------------------|------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 0      | qwerty_layer <br>(при QWERTY\_ BASE) или colemak_dhm\_ layer (иначе); BASE_L = 0 | Base             | Автоматически (базовый слой по умолчанию). На Base в тамб кластере назначены: <br> кнопка №1 — \&lt MOU_L LGUI (тап — LGUI, удержание — Mouse),<br> кнопка №2 — \&mo NUM_L (удержание — Num), <br> кнопка №3 — \&lt NAV_L RET (тап — RET, удержание — Nav), <br> кнопка №4 — \&kp SPACE, <br> кнопка №5 — \&kp BSPC, <br> кнопка №6 — \&shifty. | Переключение варианта Base (QWERTY/Colemak DHm) выполняется на этапе сборки через условную компиляцию; привязки runtime-переключения между вариантами Base отсутствуют. Возврат на Base предусмотрен на слое Mouse через \&to BASE_L (в тамб кластере кнопки №1 и №2).                                                                                                       |
| 1      | nav_layer; <br>NAV_L = 1                                                         | Nav              | Удержание \&lt NAV_L RET на Base (в тамб кластере кнопка №3; тап — RET, удержание — Nav). Дополнительно: \&lt NAV_L RET присутствует на слое Num (в тамб кластере кнопка №3; тап — RET, удержание — Nav).                                                                                                                                       | Отдельных привязок вида \&mo NAV_L/&to NAV_L не задано; основной механизм входа — \&lt NAV_L RET. Для устойчивой активации требуется удержание (не тап).                                                                                                                                                                                                                     |
| 2      | num_layer; <br>NUM_L = 2                                                         | Num              | Удержание \&mo NUM_L на Base (в тамб кластере кнопка №2). Также \&mo NUM_L присутствует на слое Nav (в тамб кластере <br>кнопка №2).                                                                                                                                                                                                            | Привязок вида \&to NUM_L не задано. Механизм \&mo предполагает работу слоя только на время удержания.                                                                                                                                                                                                                                                                        |
| 3      | adjust_layer; <br>MED_L = 3                                                      | Adjust           | 1) Условное включение (tri-layer): при одновременной активности NUM_L и NAV_L (правило if-layers = \<NUM_L NAV_L\>; then-layer = \<MED_L\>;). 2) Удержание \&mo MED_L на слое Nav (в тамб кластере кнопка №6). 3) Удержание \&mo MED_L на слое Num (в тамб кластере кнопка №6).                                                                 | Условный (tri-layer) способ предусмотрен конфигурацией и зависит от фактического одновременного удержания слоёв NUM_L и NAV_L (учитывая, что NAV_L включается через \&lt, а NUM_L — через \&mo). Прямые способы через \&mo MED_L на Nav и Num доступны постоянно при удержании соответствующей кнопки тамб кластера.                                                         |
| 4      | media_layer; <br>FUN_L = 4                                                       | Media            | Удержание \&mo FUN_L на слое Adjust (в тамб кластере <br>кнопка №3).                                                                                                                                                                                                                                                                            | Прямого входа в Media с Base не предусмотрено. Фактический путь: активировать Adjust (слой MED_L) одним из способов, затем удерживать \&mo FUN_L (тамб кластер кнопка №3 на слое Adjust).                                                                                                                                                                                    |
| 5      | mouse_layer; <br>MOU_L = 5                                                       | Mouse            | Удержание \&lt MOU_L LGUI на Base (в тамб кластере кнопка №1; тап — LGUI, удержание — Mouse). Также \&lt MOU_L LGUI присутствует на слоях Nav и Num (в тамб кластере кнопка №1). Выход на Base: \&to BASE_L (в тамб кластере кнопки №1 и №2) на слое Mouse.                                                                                     | Слой Mouse активируется только на время удержания \&lt MOU_L LGUI на Base. На слое Mouse предусмотрены две отдельные позиции возврата на Base (&to BASE_L в тамб кластере кнопка №1 и кнопка №2). Для корректной работы действий \&mkp требуется включённая поддержка pointing в конфигурации прошивки (например, через параметр сборки), иначе события мыши не формируются. |
| 6      | reserved_layer; <br>status = "disabled"                                          | Reserved (выкл.) | Не предусмотрено (привязки bindings отсутствуют).                                                                                                                                                                                                                                                                                               | Слой отключён через status = "disabled". Для использования требуется установить status = "okay" и назначить привязки включения (&mo/&to) на требуемые позиции, включая (при необходимости) кнопки тамб кластера.                                                                                                                                                             |
